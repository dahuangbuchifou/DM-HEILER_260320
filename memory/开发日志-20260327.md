## 🔍 11:34 - 第二个问题：数据库是空的

**问题：** Database Inspector 里 ticket_tasks 表存在，但里面什么都没有

**排查过程：**
```
用户："保存后也是灰色，Database Inspector 没有内容"
我："正常，因为 App 还没创建数据库..."
```

**真正的原因：** 打开 Logcat 发现没有崩溃，但数据就是没保存。

**深入排查：** 查看 TaskConfigActivity.kt 的 `saveTask()` 方法

**震惊！** 代码里赫然写着一行：
```kotlin
// TODO: 保存到数据库
// lifecycleScope.launch {
//     taskDao.insertTask(task)
// }
```

**保存逻辑被注释掉了！** 所以虽然提示"✅ 任务保存成功"，但数据根本没写入数据库！

**修复：**
```kotlin
// ✅ 保存到数据库
lifecycleScope.launch {
    try {
        val db = TaskDatabase.getDatabase(this@TaskConfigActivity)
        db.taskDao().insertTask(task)
        Toast.makeText(this@TaskConfigActivity, "✅ 任务保存成功", Toast.LENGTH_SHORT).show()
        finish()
    } catch (e: Exception) {
        Log.e(TAG, "保存任务失败", e)
        Toast.makeText(this@TaskConfigActivity, "❌ 保存失败：${e.message}", Toast.LENGTH_LONG).show()
    }
}
```

**同时添加了新字段：**
```kotlin
audienceName = viewerNames.split(",").firstOrNull()?.trim() ?: ""
selectedPrice = ticketPrice
```

**教训：** TODO 注释的代码要及时处理，否则会成为定时炸弹。

## 📋 13:19 - 第三个问题：任务列表不显示

**问题：** 保存成功了，但主界面任务列表还是空白

**排查过程：**
```
用户："数据库里有数据了，但列表没显示"
我："让我看看 MainActivity 的 loadTasks()..."
```

**又一个大坑！** MainActivity 里写的是：
```kotlin
private fun loadTasks() {
    val dummyTasks = listOf(
        TicketTask(
            id = 1,
            name = "测试抢票任务",
            concertKeyword = "刘宇宁",
            // ... 写死的假数据
        )
    )
    currentTasks.clear()
    currentTasks.addAll(dummyTasks)
    taskAdapter.submitList(currentTasks.toList())
}
```

**全是假数据！** 难怪数据库里有数据但界面不显示！

**修复：**
```kotlin
private fun loadTasks() {
    lifecycleScope.launch {
        try {
            val db = TaskDatabase.getDatabase(this@MainActivity)
            val tasks = db.taskDao().getAllTasks()
            val taskList = tasks.first()
            currentTasks.clear()
            currentTasks.addAll(taskList)
            taskAdapter.submitList(taskList)
            Log.d(TAG, "加载任务成功，共 ${taskList.size} 个任务")
        } catch (e: Exception) {
            Log.e(TAG, "加载任务失败", e)
        }
    }
}
```

**编译错误！** `lifecycleScope` 未解析...

**修复导入：**
```kotlin
import androidx.lifecycle.lifecycleScope
import kotlinx.coroutines.flow.first
```

**教训：** 假数据测试后要及时替换成真实逻辑，否则会成为"特性"。

## 🎨 14:59 - 第四个问题：显示优化

**问题：** 任务列表显示了，但格式不友好

**原始显示：**
```
关键词：刘宇宁 | 日期：2026-03-28 1774678440000 | 票档：500
```

**问题：**
- 日期是时间戳，人类看不懂
- 没有显示观众人信息
- 票档显示的是旧字段

**修复 TaskAdapter：**
```kotlin
// 优化日期格式
val dateStr = if (task.grabTime > 0) {
    val sdf = SimpleDateFormat("MM-dd HH:mm", Locale.getDefault())
    sdf.format(Date(task.grabTime))
} else {
    task.grabDate
}

// 显示观众人
val audienceStr = if (task.audienceName.isNotEmpty()) 
    task.audienceName else "未指定"

// 显示选中的票档
val priceStr = if (task.selectedPrice.isNotEmpty()) 
    task.selectedPrice else task.ticketPriceKeyword

taskDetails.text = "关键词：${task.concertKeyword} | 日期：$dateStr | 观众：$audienceStr | 票档：$priceStr"
```

**优化后显示：**
```
关键词：刘宇宁 | 日期：03-28 12:00 | 观众：张三 | 票档：888 元
```

**教训：** 用户体验藏在细节里，多花一分钟优化，用户少花十分钟理解。
