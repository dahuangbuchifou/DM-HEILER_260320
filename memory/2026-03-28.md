## 🎯 用户需求确认 (21:33)

### 大麦抢票流程（最新）
```
【预售阶段】预约 → 选择场次 → 选择票档 → 倒计时
【开票瞬间】立即预定 → 选票档 → 确认观影人 → 提交
```

### 核心需求
- **人工操作太慢**：开票瞬间需要 4 步操作，人工根本抢不到
- **需要自动化**：在开票瞬间立即自动完成 4 步操作
- **预约 + 秒杀模式**：
  1. 提前准备：App 整合所有预约信息（场次、票档、观影人）
  2. 倒计时监控：监控开票时间
  3. 瞬间执行：时间一到，自动完成 4 步操作

### UI 问题反馈
- 助手 App 可活动区域太小（红框区域）
- 建议：任务列表区域高度增加 50% 或全屏显示

### 待确认事项

## 💡 经验教训

1. **协程作用域要分清** - `suspend fun` 和 `launch {}` 块内的返回方式不同
2. **代码插入要谨慎** - 避免重复逻辑和变量未定义
3. **分屏检测要全面** - 考虑焦点不在目标 App 的情况
4. **版本时间要同步** - 三位置同时更新，保持信息一致
5. **批量修改要验证** - sed 批量替换后需逐处验证
