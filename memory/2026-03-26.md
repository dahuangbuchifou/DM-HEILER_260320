### Git 分支问题

**发现：** 本地分支是 `master`，远程分支是 `main`（GitHub 默认分支已改名）

**解决方案：**
```bash
git stash          # 暂存本地修改
git checkout main  # 切换到 main 分支
git pull origin main
git stash pop      # 恢复本地修改（可选）
```

**教训：**
1. 修改代码后必须 commit + push，否则其他设备拉不到
2. 分支名称不一致会导致 Pull 混淆
3. 有未提交修改时无法切换分支（Git 保护机制）
