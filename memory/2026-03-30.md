2. 检测信息不完整时自动点击按钮（预约抢票、确定、立即提交等）

## 09:00 - 功能实现完成

### 新增文件
- ✅ `TicketGrabbingControllerEnhanced.kt` - 增强版抢票控制器

### 核心功能

**1. 完整信息提取**
- 演出标题（支持【】、「」等格式）
- 演出时间（多场次，格式：2026-05-15 周五 19:30）
- 演出地点（体育中心、体育馆等）
- 票价档位（所有可选价格）
- 库存状态（有票/缺货登记）

**2. 自动交互流程**
- 检测信息完整性
- 不完整时自动点击：预约抢票、确定、立即提交、下一步、确认选座、配送方式、提交订单等
- 智能导航到付款界面

**3. 付款界面检测**
- 关键词：支付宝支付、微信支付、银行卡支付、付款金额、确认支付等
- 到达付款界面自动停止操作

**4. 数据模型**
```kotlin
data class CompleteConcertInfo(
    var title: String,                    // 演出标题
    var showTimes: List<ShowTimeInfo>,    // 演出时间（多场次）
    var location: String,                 // 演出地点
    var priceTiers: List<PriceTierInfo>,  // 票价档位
    var stockStatus: Map<Int, Boolean>    // 库存状态
)

data class ShowTimeInfo(
    val date: String,      // 日期：2026-05-15
    val weekday: String,   // 星期：周五
    val time: String,      // 时间：19:30
    val fullText: String
)

data class PriceTierInfo(
    val price: Int,        // 价格：580
    val isSoldOut: Boolean, // 是否售罄
    val fullText: String
)
```

### 工作流程
```
1. 检测分屏状态 → 2. 开始截屏（2FPS） → 3. 提取完整演出信息
→ 4. 检查信息完整性 → 5. 自动导航获取更多信息（如需要）
→ 6. 选择票档 → 7. 选择观众 → 8. 提交订单（自动导航）
→ 9. 到达付款界面 → 停止 → 10. 输出完整信息报告
```

### 状态机更新
```
IDLE → SEARCHING → INFO_EXTRACTING → PRICE_SELECTING 
→ AUDIENCE_SELECTING → SUBMITTING → PAYMENT_PAGE → COMPLETED
```
