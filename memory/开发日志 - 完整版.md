## 📅 2026-03-25 - 编译修复与真机测试

**主题：** 24 个编译错误的攻坚战

### 挑战

打开 Android Studio，迎接我们的是 **24 个编译错误**：

```
❌ Unresolved reference: GLOBAL_ACTION_BACK
❌ Type mismatch: inferred 'String' but 'EventType' expected
❌ Object 'AccessibilityUtils' is not an instance
❌ kapt is disabled, cannot process Room entities
...
```

### 战斗过程

#### 第一回合：配置错误（6 个错误）

**问题 1：** Room 数据库无法编译
```
Error: kapt is disabled, cannot process Room entities
```

**解决：** 启用 kapt 插件
```gradle
plugins {
    id 'kotlin-kapt'
}
```

**问题 2：** Kotlin 版本不一致
```
Error: Kotlin stdlib version mismatch
```

**解决：** 统一版本为 1.9.25
```gradle
implementation "org.jetbrains.kotlin:kotlin-stdlib:1.9.25"
```

#### 第二回合：类型错误（10 个错误）

**问题：** 枚举和字符串混用
```kotlin
// ❌ 错误
eventType = "TASK_START"

// ✅ 正确
eventType = EventType.TASK_START
```

**修复文件：**
- `TicketGrabbingAccessibilityService.kt` (10 处)

#### 第三回合：API 调用错误（4 个错误）

**问题：** Android SDK 常量无法解析
```kotlin
// ❌ 编译错误
performGlobalAction(GLOBAL_ACTION_BACK)
```

**解决：** 使用整数值
```kotlin
// ✅ 直接写值
performGlobalAction(1)  // GLOBAL_ACTION_BACK = 1
```

#### 第四回合：对象实例化错误（2 个错误）

**问题：** Kotlin object 不能实例化
```kotlin
// ❌ 错误
val utils = AccessibilityUtils()

// ✅ 正确
val enabled = AccessibilityUtils.isServiceEnabled()
```

## 📅 2026-03-26 - 字段缺失与分支管理

**主题：** 当代码不在一个频道上

### 早晨的惊吓

编译报错：
```
❌ Unresolved reference 'selectedPrice' (第 864 行)
❌ Unresolved reference 'audienceName' (第 914 行)
```

**原因：** 
- `TicketGrabbingAccessibilityService.kt` 使用了这两个字段
- 但 `TicketTask.kt` 里根本没有定义！

**修复：**
```kotlin
// TicketTask.kt
val audienceName: String = "",   // 步骤 3：自动选择观演人
val selectedPrice: String = ""   // 步骤 2：选择票档
```

### Git 分支混乱

**发现问题：**
```bash
$ git branch
* master

$ git remote -v
origin  github.com:dahuangbuchifou/DM-HEILER_260320.git
# 但远程默认分支是 main！
```

**解决：**
```bash
git stash          # 暂存修改
git checkout main  # 切换到 main
git pull origin main
git stash pop      # 恢复修改
```

**教训：**
1. 分支名称要统一（main 是 GitHub 默认）
2. 修改必须 commit + push，否则别人看不到
3. 有未提交修改时无法切换分支（Git 保护机制）

### 字段设计优化

| 字段 | 用途 | 默认值 |
|------|------|--------|
| `viewerNames` | 数据库存储（逗号分隔多人） | - |
| `audienceName` | UI 自动化（单个姓名） | `""` |
| `ticketPriceKeyword` | 票价搜索关键词 | - |
| `selectedPrice` | 选中的票价 | `""` |

**设计思路：** 
- 数据库字段保持灵活性（支持多人）
- 自动化字段简化逻辑（单人快速选择）

**用户：** "保存后也是灰色，Database Inspector 没有内容"

**排查过程：**
1. 检查 Logcat - 无崩溃
2. 检查数据库 - 表结构正确但无数据

```kotlin
// TaskConfigActivity.kt 第 401 行
// TODO: 保存到数据库
// lifecycleScope.launch {
//     taskDao.insertTask(task)
// }
```

**保存逻辑被注释掉了！**

**修复：**
```kotlin
// ✅ 保存到数据库
lifecycleScope.launch {
    try {
        val db = TaskDatabase.getDatabase(this@TaskConfigActivity)
        db.taskDao().insertTask(task)
        Toast.makeText(this, "✅ 任务保存成功", Toast.LENGTH_SHORT).show()
        finish()
    } catch (e: Exception) {
        Log.e(TAG, "保存任务失败", e)
        Toast.makeText(this, "❌ 保存失败：${e.message}", Toast.LENGTH_LONG).show()
    }
}
```

**同时添加新字段：**
```kotlin
audienceName = viewerNames.split(",").firstOrNull()?.trim() ?: ""
selectedPrice = ticketPrice
```

**教训：** TODO 注释的代码要及时处理，否则会成为定时炸弹。

### 13:19 - 第三个问题：任务列表不显示

**用户：** "数据库里有数据了，但列表没显示"

**排查：** 查看 MainActivity 的 `loadTasks()`

```kotlin
private fun loadTasks() {
    val dummyTasks = listOf(
        TicketTask(
            id = 1,
            name = "测试抢票任务",
            concertKeyword = "刘宇宁",
            // ... 写死的假数据
        )
    )
    currentTasks.clear()
    currentTasks.addAll(dummyTasks)
}
```

**全是假数据！** 难怪不显示真实数据！

**修复：**
```kotlin
private fun loadTasks() {
    lifecycleScope.launch {
        val db = TaskDatabase.getDatabase(this@MainActivity)
        val tasks = db.taskDao().getAllTasks()
        val taskList = tasks.first()
        currentTasks.clear()
        currentTasks.addAll(taskList)
        taskAdapter.submitList(taskList)
    }
}
```

**编译错误！** `lifecycleScope` 未解析...

**添加导入：**
```kotlin
import androidx.lifecycle.lifecycleScope
import kotlinx.coroutines.flow.first
```

**教训：** 假数据测试后要及时替换，否则会成为"特性"。
