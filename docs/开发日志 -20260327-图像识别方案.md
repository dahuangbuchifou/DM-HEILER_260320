## 🕐 22:10 - 作用域错误修复

**错误：** `chineseRecognizer`、`latinRecognizer` 在 `close()` 中未解析

**原因：** 这些变量定义在 `companion object` 中，但 `close()` 是实例方法

**修复：** 将识别器提升为类成员变量
```kotlin
class ScreenAnalyzer {
    // 类成员变量（而非 companion object）
    private val chineseRecognizer = TextRecognition.getClient(...)
    private val latinRecognizer = TextRecognition.getClient(...)
    
    fun close() {
        chineseRecognizer.close()
        latinRecognizer.close()
    }
}
```

**版本时间：** 2026-03-27 22:10

## 🎯 关键决策点

### 决策 1：分屏 vs 悬浮窗
**当前：** 分屏模式  
**问题：** 助手 App 滑动受限  
**下一步：** 悬浮窗模式（System Alert Window）

### 决策 2：本地 OCR vs 云端 OCR
**当前：** 本地 ML Kit  
**优势：** 免费、快速、离线  
**劣势：** 识别率有限  
**备选：** 百度 OCR（云端，识别率高）

### 决策 3：纯图像识别 vs 混合方案
**当前：** 纯图像识别  
**推荐：** 混合方案（图像识别 + 无障碍服务）
- 图像识别：分析界面、计算坐标
- 无障碍服务：执行点击、验证结果

## 📖 自动测试方案

**问题：** 本地内存不足，无法运行完整测试

**解决方案：**

### 方案 A：GitHub Actions（推荐）
```yaml
name: Android Tests
on: [push]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Run Tests
        run: ./gradlew test
```

**优势：** 免费、集成 Git、自动触发  
**劣势：** 无法真机测试

### 方案 B：Firebase Test Lab
```bash
gcloud firebase test android run \
  --type instrumentation \
  --app app-debug.apk \
  --test app-debug-androidTest.apk \
  --device model=cheetah,version=33
```

**优势：** 真机测试、多设备并行  
**劣势：** 免费额度有限（每天 5 次）

### 方案 C：本地模拟器 + 自动化脚本
```bash
# 启动模拟器
emulator -avd Pixel_6_API_33

# 运行测试
adb shell am instrument -w com.damaihelper.test

# 自动截图对比
adb shell screencap -p /sdcard/screen.png
adb pull /sdcard/screen.png
```

**优势：** 完全控制、可调试  
**劣势：** 占用本地资源

### 推荐方案
**短期：** GitHub Actions（CI/CD）  
**中期：** Firebase Test Lab（真机验证）  
**长期：** 自建测试农场（多台旧手机）
