## 🧪 测试用例

### 1. API 抢票测试

#### TC-API-001: 正常抢票流程

**前置条件**:
- 已登录大麦账号
- 有效的访问令牌
- 网络正常

**测试步骤**:
1. 创建抢票任务
2. 设置抢票时间为未来 1 分钟
3. 启动抢票
4. 观察日志

**预期结果**:
- 抢票成功或返回明确错误
- 响应时间 < 3s
- 日志完整

**优先级**: P0

---

#### TC-API-002: 令牌过期自动刷新

**前置条件**:
- 访问令牌已过期
- 刷新令牌有效

**测试步骤**:
1. 手动使令牌过期
2. 启动抢票
3. 观察是否自动刷新

**预期结果**:
- 自动刷新令牌
- 抢票继续执行

**优先级**: P1

---

#### TC-API-003: 熔断机制测试

**前置条件**:
- 模拟连续失败场景

**测试步骤**:
1. 模拟 API 返回错误
2. 连续触发 5 次失败
3. 观察是否切换策略

**预期结果**:
- 第 5 次失败后触发熔断
- 自动切换到 UI 自动化策略

**优先级**: P1
