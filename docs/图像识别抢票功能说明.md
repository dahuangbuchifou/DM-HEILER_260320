## 📋 使用流程

### 1. 准备工作

**设备要求：**
- Android 10+（支持 MediaProjection API）
- 支持分屏功能
- 已安装大麦 App

**权限配置：**
- 截屏权限（首次使用会提示）
- 无障碍服务权限（用于执行点击）

### 2. 开启分屏

**手动分屏步骤：**
1. 打开抢票助手 App
2. 点击"最近任务"按钮（方形键）
3. 长按抢票助手窗口
4. 选择"分屏"
5. 在另一半屏幕打开大麦 App

**效果：**
```
┌─────────────────┬─────────────────┐
│  抢票助手 (左)   │   大麦 App (右)  │
│                 │                 │
│  📸 截屏分析     │   🎫 演出详情    │
│  🔍 图像识别     │   🎫 票档选择    │
│  👆 模拟点击     │   👤 观众选择    │
└─────────────────┴─────────────────┘
```

### 3. 创建抢票任务

1. 在抢票助手中点击"➕ 手动添加任务"
2. 填写任务信息：
   - **演出关键词**：刘宇宁
   - **抢票日期**：2026-03-28
   - **抢票时间**：12:00
   - **票档价格**：888 元
   - **观众人姓名**：张三
3. 保存任务

### 4. 开始抢票

1. 确保已分屏（助手 + 大麦）
2. 点击任务的"开始抢票"按钮
3. 观察助手操作过程

**自动流程：**
```
1. 检测分屏状态 ✅
2. 开始截屏（1 FPS） ✅
3. 搜索演出（需手动输入关键词） ⚠️
4. 选择票档（自动识别并点击） ✅
5. 选择观众（自动识别并点击） ✅
6. 提交订单（自动点击购买按钮） ✅
```

## 🐛 常见问题

### Q1: 无法开始截屏
**A:** 首次使用需要授权截屏权限
- 点击"开始抢票"后会弹出授权窗口
- 点击"立即开始"即可

### Q2: 分屏检测失败
**A:** 确保两个 App 同时可见
- 检查是否真的分屏（不是一前一后）
- 检查屏幕宽度是否正常

### Q3: 识别不到票档
**A:** 可能原因：
- 票档文字不清晰
- 票档已售罄（灰色）
- 截图时机不对

**解决：**
- 调整截屏频率（提高 FPS）
- 手动刷新页面
- 检查票档是否可售

### Q4: 点击位置不准确
**A:** 可能原因：
- 分屏比例变化
- 屏幕密度不同

**解决：**
- 重新分屏
- 重启 App

## 📊 性能优化

### 功耗控制
- **截屏频率**：默认 1 FPS，可根据需要调整
- **识别区域**：只识别关键区域（减少计算量）
- **智能休眠**：页面不变时不重复识别

### 识别速度
- **本地识别**：ML Kit 本地处理（<100ms）
- **并行处理**：截屏和识别异步进行
- **缓存结果**：界面不变时复用识别结果
