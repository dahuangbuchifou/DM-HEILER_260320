## 🚀 快速开始

### 1. 创建抢票任务

```kotlin
val task = TicketTask(
    concertKeyword = "【烟台】李宗盛 2026『有歌之年』巡回演唱会",
    grabDate = "2026-03-30",
    grabTime = 1711785600000L,
    sessionId = "2026-05-15-19:30",  // 指定场次
    priceTiers = "1580,1280,880",     // 多票档备选（从高到低）
    selectedPrice = "1580",           // 首选票档
    audienceName = "张三",
    audienceIndex = 1,                // 默认第一个观众
    grabMode = "snap"                 // 秒杀模式
)
```

### 2. 开始抢票

```kotlin
// 创建增强版控制器
val controller = TicketGrabbingControllerEnhanced(
    accessibilityService = this,
    screenCapture = screenCaptureService,
    analyzer = screenAnalyzer
)

// 开始抢票
controller.startGrabbing(task)

// 监听状态
controller.stateFlow.collect { state ->
    when (state) {
        GrabbingState.INFO_EXTRACTING -> println("📋 提取信息...")
        GrabbingState.PRICE_SELECTING -> println("🎫 选票档...")
        GrabbingState.AUDIENCE_SELECTING -> println("👤 选观众...")
        GrabbingState.SUBMITTING -> println("📝 提交订单...")
        GrabbingState.PAYMENT_PAGE -> println("💳 到达付款界面！")
        GrabbingState.COMPLETED -> println("✅ 完成！")
        else -> {}
    }
}
```

## 🔄 自动交互流程

### 支持的自动点击按钮

```kotlin
ACTION_BUTTONS = [
    "预约抢票", "确定", "立即提交", "下一步",
    "确认选座", "配送方式", "提交订单",
    "立即购买", "去支付", "确认下单"
]
```

### 工作流程

```
1. 检测分屏状态 ✅
   ↓
2. 开始截屏（2FPS） ✅
   ↓
3. 提取完整演出信息 ✅
   - 标题、时间、地点、价格、库存
   ↓
4. 检查信息完整性 ✅
   - 完整 → 继续
   - 不完整 → 自动点击按钮导航
   ↓
5. 选择票档（支持多票档备选） ✅
   ↓
6. 选择观众（支持索引） ✅
   ↓
7. 提交订单（自动导航） ✅
   - 循环点击按钮直到付款界面
   ↓
8. 到达付款界面 → 停止 ✅
   ↓
9. 输出完整信息报告 ✅
```

## ⚙️ 配置选项

### 截屏频率

```kotlin

screenCapture.startCapture(fps = 2)

// 可调整：
// - 1 FPS (每秒 1 帧) - 省电
// - 2 FPS (每秒 2 帧) - 平衡（推荐）
// - 5 FPS (每秒 5 帧) - 极速（耗电）
```

### 多票档备选

```kotlin
// 配置多个票档，从高到低排列
priceTiers = "1580,1280,880"

// 系统会按顺序尝试每个票档，直到找到有票的
```

## ⚠️ 注意事项

### 必需条件
1. ✅ Android 10+（支持 MediaProjection API）
2. ✅ 分屏模式（助手 + 大麦）
3. ✅ 截屏权限授权
4. ✅ 无障碍服务权限

### 使用建议
1. 确保大麦页面信息完整显示
2. 保持网络畅通（OCR 识别）
3. 观察日志输出确认状态
4. 到达付款界面后手动完成支付

### 支付安全（重要！）
- ⚠️ **到达付款界面后必须手动完成支付**
- ⚠️ 自动操作在付款界面自动停止
- ⚠️ 不存储任何支付信息

## 🐛 常见问题

### Q1: 信息提取不完整怎么办？

**A:** 增强版会自动尝试：
1. 多次截屏识别（最多 10 次）
2. 自动点击按钮导航（预约、确定、下一步等）
3. 如果仍然不完整，会输出已提取的信息并继续流程

**手动解决：**
- 确保大麦页面信息完整显示
- 检查网络连接
- 重启分屏模式

### Q2: 自动点击不生效？

**A:** 可能原因：
- 无障碍服务未开启
- 分屏模式未正确检测
- 按钮文字识别失败

**解决：**
- 检查无障碍服务权限
- 重新分屏
- 查看日志确认识别结果

### Q3: 如何知道到达了付款界面？

**A:** 观察日志输出：
```
✅ 检测到付款界面关键词：支付宝支付
✅ 到达付款界面，停止自动操作
```

状态会变为 `PAYMENT_PAGE`。

### Q4: 多票档备选如何使用？

**A:** 在任务中配置 `priceTiers`：
```kotlin
priceTiers = "1580,1280,880"  // 从高到低排列
```

系统会按顺序尝试每个票档，直到找到有票的。
