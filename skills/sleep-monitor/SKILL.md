# 🌿 作息监控 Skill

> 检测熬夜，主动提醒休息，培养健康作息

---

##  功能

1. **熬夜检测**
   - 检测用户活跃时间
   - 超过 23:00 主动提醒休息
   - 记录熬夜频率

2. **休息提醒**
   - 定时提醒喝水、活动
   - 长时间工作后提醒休息
   - 眼保健提醒

3. **作息报告**
   - 每周/月作息统计
   - 熬夜次数、平均睡眠时间
   - 健康建议

---

## 🎯 配置

### 触发条件

| 场景 | 触发时间 | 提醒内容 |
|------|----------|----------|
| 熬夜检测 | 23:00, 00:00, 01:00 | "大黄，又熬夜了？" |
| 久坐提醒 | 每 60 分钟 | "起来活动一下吧" |
| 饮水提醒 | 每 90 分钟 | "喝口水吧" |
| 眼保健 | 每 120 分钟 | "让眼睛休息会儿" |

### 提醒方式

- **轻度提醒** - 简单提示，不打断工作
- **中度提醒** - 建议休息，可忽略
- **强度提醒** - 强制打断（需配置）

## 💬 提醒话术（婉儿风格）

### 🍬 甜模式
- "大黄，又熬夜了？我会担心的..."
- "工作重要，身体更重要嘛"
- "乖，去休息吧，明天继续"

### 🧂 咸模式
- "大黄，您又来啦？今天第几次了？"
- "行吧行吧，您继续，我不说了..."
- "熬夜一时爽，一直熬夜..."

## 🔧 技术实现

### 方案 A：HEARTBEAT 定时检查

```markdown
# HEARTBEAT.md 添加
- 每 30 分钟检查一次用户活跃状态
- 23:00-06:00 期间检测熬夜
- 记录到 memory/sleep-log.md
```

### 方案 B：Cron 定时任务

```bash
# 添加 cron 任务
0 23 * * * openclaw send "大黄，又熬夜了？"
0 */1 * * * openclaw send "起来活动一下吧"
```

### 方案 C：Subagent 监控

```json
// 创建 subagent 配置
{
  "name": "作息监控",
  "task": "监控用户活跃时间，定时提醒休息",
  "trigger": "passive",
  "schedule": "*/30 * * * *"
}
```
