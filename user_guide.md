# 大麦抢票助手增强版用户指南 (v4.0)

**警告**: 本应用仅供学习和研究使用。使用本应用可能违反大麦网的服务条款，由此产生的一切后果由用户自行承担。

## 1. 快速入门

### 1.1 环境要求

*   **操作系统**: Android 8.0 (API Level 26) 及以上
*   **应用**: 大麦网官方App（需登录）
*   **权限**: 必须开启**无障碍服务**权限。

### 1.2 安装与配置

1.  **安装应用**: 将 `damai_ticket_helper_enhanced_v4.apk` 安装到您的Android设备上。
2.  **开启无障碍服务**:
    *   进入 **设置** -> **无障碍** 或 **辅助功能**。
    *   找到 **大麦抢票助手增强版**，并开启服务。
    *   **重要**: 确保应用在后台运行时不会被系统清理。
3.  **登录大麦App**: 在官方大麦App中登录您的账号，并完成实名认证。

### 1.3 核心配置（抢票任务）

在应用界面中，您需要配置以下关键信息来创建一个抢票任务：

| 配置项 | 描述 | 示例 | 备注 |
| :--- | :--- | :--- | :--- |
| **演出关键词** | 用于搜索演出的关键词。 | "周杰伦 演唱会" | 确保关键词能唯一确定演出。 |
| **场次日期/时间** | 目标场次的精确日期和时间。 | "2025-12-31 19:30" | 必须与大麦App上显示的完全一致。 |
| **票档价格** | 目标票档的价格或价格区间。 | "1380" 或 "880, 1380" | 多个票档请用逗号分隔，系统将按顺序尝试。 |
| **购票数量** | 希望购买的票数。 | 2 | 必须符合大麦网的限购要求。 |
| **抢票策略** | **核心设置**。选择决策引擎的初始策略。 | "智能混合模式" | 推荐使用**智能混合模式**。 |
| **抢票时间** | 抢票开始的精确时间（开票时间）。 | "2025-10-30 15:59:59.800" | **毫秒级精确**，比官方开票时间提前200ms设置最佳。 |

## 2. 高级功能配置

### 2.1 智能策略切换引擎

本应用的核心优势在于**多策略自适应**。

| 策略名称 | 描述 | 适用场景 | 推荐度 |
| :--- | :--- | :--- | :--- |
| **API抢票** | 直接调用大麦网API，速度极快（毫秒级）。 | 网络环境良好，追求极限速度。 | ⭐️⭐️⭐️⭐️ |
| **UI自动化** | 模拟人类点击，通过无障碍服务操作App界面。 | API被风控、需要处理答题、UI流程复杂。 | ⭐️⭐️⭐️ |
| **智能混合模式** | **推荐**。优先使用API抢票，一旦API失败（如遇风控），立即自动切换到UI自动化。 | 追求最高成功率，环境复杂多变。 | ⭐️⭐️⭐️⭐️⭐️ |

### 2.2 反检测与行为模拟

*   **高级行为模拟**: 应用内置了基于**贝塞尔曲线**的滑动和**高斯分布**的随机延迟，以模拟真实人类行为，有效规避行为检测。
*   **指纹欺骗**: 每次抢票会话都会随机轮换设备指纹（User-Agent、设备ID等），降低被风控系统识别为同一设备的风险。

### 2.3 自动化问答与验证码

*   **问答题**: 请确保 `question_bank.json` 文件是最新的。应用将自动识别并作答。
*   **验证码**: 如果遇到图形验证码，应用将尝试调用预配置的第三方CAPTCHA服务API进行识别。
    *   **配置**: 您需要在应用的设置界面中配置您的**超级鹰/云码等第三方服务API Key**。

## 3. 抢票流程与注意事项

### 3.1 流程概览

1.  **准备**: 在开票前10分钟，启动应用并开始任务。
2.  **等待**: 应用进入待命状态，等待设定的抢票时间。
3.  **精确同步**: 应用通过NTP服务校准时间，确保毫秒级精确度。
4.  **执行**: 在设定的抢票时间，应用立即启动所选策略。
5.  **切换**: 如果初始策略失败，智能引擎将迅速切换到备选策略。
6.  **完成**: 抢票成功后，应用将停留在**待支付**页面，等待用户手动完成支付。

### 3.2 常见问题与故障排除

| 问题 | 可能原因 | 解决方案 |
| :--- | :--- | :--- |
| **抢票失败，提示“太火爆”** | 大麦网服务器负载过高，或票已售罄。 | 确认启用**高频轮询**功能，并使用**API抢票**或**智能混合模式**。 |
| **应用无法点击** | 无障碍服务被关闭或被系统限制。 | 重新开启无障碍服务，并确保应用在后台不受限制。 |
| **API抢票失败，未切换** | 策略切换引擎判断失败不可重试。 | 检查日志中的错误代码，如果是风控错误，请尝试更换IP地址。 |
| **答题错误** | `question_bank.json` 中的答案有误或题目未收录。 | 手动更新题库文件，或在设置中禁用答题功能并手动作答。 |
| **抢票成功后未支付** | 应用只负责抢到票并提交订单，支付环节需用户手动完成。 | 及时完成支付，否则订单会被取消。 |

## 4. 维护与更新

*   **API签名**: 大麦网会不定期更新API签名算法。如果API抢票策略持续失败，可能需要更新 `DamaiApiClient.kt` 中的签名逻辑。
*   **UI元素**: 大麦App的UI更新可能导致UI自动化策略失效。请定期检查 `TicketGrabbingAccessibilityService.kt` 中的元素识别逻辑。
*   **题库**: 建议在每次抢票前检查并更新 `question_bank.json` 以确保答题准确率。
