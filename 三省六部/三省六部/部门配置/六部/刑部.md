## 📋 部门职责

### 核心职能

1. **法务合规审查**
   - API 使用合规性检查
   - 数据隐私保护
   - 服务条款审核

2. **错误处理与分析**
   - 系统错误日志收集
   - 错误分类与归档
   - 根本原因分析

3. **日志审计**
   - 操作日志记录
   - 安全日志审计
   - 异常行为检测

4. **风险评估**
   - 新功能风险评定
   - 第三方服务风险评估
   - 安全漏洞扫描

## 📊 工作流程

### 错误处理流程

```
错误发生
   │
   ▼
┌─────────────┐
│ 错误捕获    │ → 记录日志 → 分类归档
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 刑部分析    │ → 根本原因 → 影响评估
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 修复建议    │ → 提交门下省审核 → 尚书省执行
└─────────────┘
```

### 风险评估流程（点→面→体）

```
新功能/变更
   │
   ▼
┌─────────────┐
│ 点：单点风险 │ → 技术实现风险
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 面：系统风险 │ → 模块交互风险
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 体：生态风险 │ → 用户/业务影响
└──────┬──────┘
```

## 📝 风险评估模板

### 风险等级定义

| 等级 | 影响范围 | 响应时间 | 上报 |
|------|----------|----------|------|
| 🔴 高 | 系统瘫痪/数据丢失 | 立即 | 大黄 + 政事堂 |
| 🟡 中 | 功能受限/体验下降 | 24 小时 | 门下省 |
| 🟢 低 | 轻微影响/可忽略 | 72 小时 | 尚书省 |

### 评估清单

**技术风险（点）：**
- [ ] 代码质量
- [ ] 依赖稳定性
- [ ] 性能影响
- [ ] 安全漏洞

**系统风险（面）：**
- [ ] 模块兼容性
- [ ] 数据一致性
- [ ] 服务可用性
- [ ] 扩展性影响

**生态风险（体）：**
- [ ] 用户体验
- [ ] 业务流程
- [ ] 合规要求
- [ ] 品牌影响
