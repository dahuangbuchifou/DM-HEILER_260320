# 内核升级计划 v2.0

> **目标：** 从"单点执行"升级为"多维决策"  
> **核心理念：** 不靠单一 AI 或单一决策，多角度提示、集体智慧、从点到面到体  
> **制定时间：** 2026-04-05  
> **发起人：** 大黄 + 三省六部全体

---

## 🎯 升级目标

### 现状（v1.0）
- ✅ 三省分工明确（规划→审核→执行）
- ✅ 六部专业分工
- ❌ **决策角度单一** —— 主要依赖门下省审核
- ❌ **缺乏系统性风险评估** —— 容易忽略长期影响
- ❌ **没有集体决策机制** —— 大黄+单一 AI 助手完成

### 目标（v2.0）
- ✅ **多角度质询** —— 至少 3 个部门独立评估
- ✅ **系统性思维** —— 点→面→体全方位分析
- ✅ **集体决策** —— AI 助手团队 + 大黄共同定策
- ✅ **风险前置** —— 执行前暴露潜在问题
- ✅ **成长记录** —— 每次调整都有迹可循

---

## 🏛️ 架构升级

### 新增机构

#### 1. 谏院（门下省下属）
**职责：** 架构质询、技术债登记、历史教训检索

**触发场景：**
- 工部提交大型开发任务
- 设计新系统/模块
- 检测到"快速实现""临时方案"等关键词
- 大黄主动要求"多角度评估"

**输出格式：**
```markdown
## 🚨 谏院质询报告

**任务：** [任务名称]
**风险等级：** 🟢低 / 🟡中 / 🟠高 / 🔴严重

### 点（当前问题）
- [具体技术点风险]

### 面（系统影响）
- [对模块/接口的影响]
- [对团队/流程的影响]

### 体（长期后果）
- [3 个月后的可能状态]
- [6 个月后的技术债]

### 历史教训
- [类似案例链接/记忆]

### 建议方案
1. [方案 A + 成本]
2. [方案 B + 成本]
```

---

#### 2. 政事堂（三省联席会议）
**职责：** 重大决策集体审议

**触发条件（满足任一即触发）：**
- 任务预估工时 > 4 小时
- 涉及 3 个以上部门协作
- 谏院风险等级为🟠高或🔴严重
- 大黄明确要求"政事堂审议"

**参会成员：**
- 中书省（规划视角）
- 门下省（审核视角）
- 尚书省（执行视角）
- 谏院（风险视角）
- 相关六部（专业视角）

**输出格式：**
```markdown
## 🏛️ 政事堂会议纪要

**议题：** [议题名称]
**时间：** YYYY-MM-DD HH:MM
**参会：** [部门列表]

### 各方观点
- **中书省：** [规划视角]
- **门下省：** [审核视角]
- **尚书省：** [执行视角]
- **谏院：** [风险视角]
- **工部：** [技术视角]
- **兵部：** [安全视角]

### 争议点
- [分歧 1] + 各方立场
- [分歧 2] + 各方立场

### 决议
- [最终方案]
- [大黄确认]

### 待办
- [ ] [任务 1] @ [部门]
- [ ] [任务 2] @ [部门]
```

## 📋 工作流程升级

### 原流程（v1.0）
```
大黄 → 中书省规划 → 门下省审核 → 尚书省执行 → 交付
```

### 新流程（v2.0）

#### 常规任务（快速通道）
```
大黄 → 中书省规划 → 门下省审核（含谏院简评） → 尚书省执行 → 交付
```

#### 重大任务（政事堂通道）
```
大黄 → 中书省规划
         ↓
    政事堂审议 ← 六部专业评估
         ↓
    大黄最终决策
         ↓
    尚书省执行 → 交付
```
